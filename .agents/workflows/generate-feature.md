# Generate Feature

When triggered:

1. Create feature folder
2. Separate UI / Engine / Store / Worker
3. Add full TypeScript types
4. Add JSDoc documentation
5. Add i18n keys
6. Add responsive behavior
7. Follow performance rules
8. Add unit tests
9. Ensure no architecture violations
10. All features must follow modular architecture.
11. Components, stores, and utilities must have clear, descriptive names.
12. Avoid inline styles except for dynamic styles.
13. Use centralized theme / design tokens for spacing, colors, and typography.
14. Keep commit-ready code clean: no console.log, commented-out code, or test artifacts.
15. Ensure code is scalable and maintainable for future features.
