# Internationalization Rule

- No hardcoded UI text
- Use next-intl
- All strings via translation keys
- Support dynamic locale switching
- Support RTL
- Format time and numbers via Intl API
* All UI strings must be i18n-ready.
* Use `next-i18next` or preferred i18n library.
* Do not hardcode strings in components.
* Provide placeholder translations for all supported languages.
* Keep translation keys consistent and organized per feature.

Default locale: en
